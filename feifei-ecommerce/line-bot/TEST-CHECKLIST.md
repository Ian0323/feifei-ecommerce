# LINE Bot Test Checklist

> 用途：這份清單用來做 `小非` 的真實端到端驗收。  
> 原則：先驗一對一，再驗群組，最後驗邊界情境。

---

## 1. 測試前確認

- [ ] `.env` 已建立並填入真實值
- [ ] `npm start` 可啟動
- [ ] `http://localhost:3000/health` 正常回 `{ "ok": true }`
- [ ] LINE Developers Console 已填入正確 Webhook URL
- [ ] `Use webhook` 已開啟
- [ ] LINE Verify 成功

---

## 2. 一對一私訊測試

### 2.1 FAQ 類

- [ ] 問「三款差在哪裡？」可得到正確回覆
- [ ] 問「怎麼下單？」可得到正確回覆
- [ ] 問「價格多少？」可得到合理回覆

### 2.2 產品推薦類

- [ ] 問「最近壓力很大，淨化適合嗎？」可導到淨化
- [ ] 問「約會前適合哪一款？」可導到貴人
- [ ] 問「最近業績卡卡，哪款適合？」可導到招財

### 2.3 轉人工類

- [ ] 問「我要退款」時，bot 不亂答，會引導找 feifei
- [ ] 問「我沒收到貨」時，bot 會引導找 feifei
- [ ] 問敏感身心狀況時，bot 不做不當建議

---

## 3. 群組測試

### 3.1 Mention / 關鍵字

- [ ] 在群組中 `@小非`，bot 會回覆
- [ ] 提到 `淨化`、`貴人`、`招財`，bot 可適度回覆
- [ ] 提到 `怎麼買`、`下單`、`價格`，bot 可適度回覆

### 3.2 一般聊天

- [ ] 一般聊天時 bot 不會每句都插話
- [ ] bot 回覆句數維持簡短，不洗版
- [ ] bot 不會搶 feifei 的戲份

### 3.3 節流與靜音

- [ ] 連續觸發多次後，bot 會進入冷卻
- [ ] 一小時內不會超過預設回覆上限
- [ ] 23:00-08:00 間，一般訊息不會亂回
- [ ] 深夜若明確 @ 小非，行為符合目前設定

---

## 4. 錯誤與觀察

- [ ] Server console 無明顯報錯
- [ ] LINE 後台 webhook delivery 無持續失敗
- [ ] 回覆延遲在可接受範圍
- [ ] 沒有出現空白訊息或重複訊息
