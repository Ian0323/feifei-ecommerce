# Landing Page Brief

> 用途：此文件定義 `芳香噴霧 landing page` 的第一版交付規格。  
> 讀者：Ian、Codex、Claude Code、Gemini CLI。  
> 原則：第一版先求可上線、可導購、可收單，不追求過度炫技。

---

## 1. 這一版要解決的問題

目前香霧產品沒有一個可直接承接 IG 流量的品牌頁。  
第一版 landing page 要先解決三件事：

1. 讓第一次進站的人快速理解 `這是什麼產品`
2. 讓人快速看懂 `三款差異`
3. 讓人可以直接 `下單或加入 LINE 詢問`

---

## 2. 第一版定位

這一版不是形象大片，也不是完整品牌官網。  
這一版是 `品牌感足夠的轉換型頁面`。

### 優先順序

1. 清楚
2. 信任感
3. 手機閱讀順暢
4. CTA 明確
5. 視覺一致
6. 動畫與華麗效果

### 不建議做法

- 不要依賴重動畫
- 不要首頁直接塞太長的情緒文案
- 不要過度神秘化，導致使用者看不懂
- 不要讓 CTA 埋太深

---

## 3. 目標受眾假設

根據 README 與既有分析，第一版先以以下受眾為主：

- 已追蹤 feifei 的 IG 粉絲
- 對香氣、儀式感、能量感興趣的女性
- 年齡約 25-45 歲
- 多數從手機進站
- 對 feifei 本人有基本信任

第一版不需要同時滿足所有宮廟信眾語境。  
若後續確認宮廟通路是主力，再做第二版調整。

---

## 4. 第一版資訊架構

建議結構如下：

1. `Hero`
   - 一句主標
   - 一句副標
   - 主 CTA：立即下單
   - 次 CTA：加入 LINE / 了解更多
2. `品牌價值`
   - 說明這不是一般香氛，而是帶有情緒與儀式感的日常使用品
3. `三款產品`
   - 淨化
   - 貴人
   - 招財
   - 每款都要有：
     - 圖片
     - 一句核心感受
     - 適合情境
     - 價格
4. `怎麼使用`
   - 什麼時候噴
   - 適合誰
   - 簡單使用方式
5. `品牌故事 / feifei 推薦理由`
   - 短版即可
6. `下單方式`
   - Google 表單
   - 匯款流程
   - 出貨說明
7. `FAQ`
   - 幾個最常見問題即可
8. `最終 CTA`
   - 表單
   - LINE
   - IG

---

## 5. 文案方向

### 語氣原則

- 溫柔
- 親近
- 有感受但不玄
- 像主理人在推薦自己真的會用的東西
- 不要像大品牌廣告

### Hero 文案方向

先不要用太多抽象句。  
第一版建議是「一句情緒價值 + 一句產品說明」。

#### 建議格式

- 主標：
  - `每天一噴，讓好事慢慢靠近你`
  - 或 `用香氣，照顧每天的自己`
- 副標：
  - `M+ feifei 水晶香霧，陪你在日常裡找回安定、吸引力與好能量。`

### 產品卡文案格式

每款維持固定結構：

- 名稱
- 一句話定位
- 適合什麼時候使用
- 價格
- CTA

### FAQ 建議題目

- 三款差在哪裡？
- 怎麼下單？
- 付款方式是什麼？
- 多久出貨？
- 可以先詢問再買嗎？

## 6. 素材清單

以下素材到齊後，AI 才能高效率做第一版：

### 必備

- 三款產品名稱
- 三款產品售價
- 每款 1-2 張可用圖片
- 品牌名稱顯示方式
- Google 表單連結
- LINE 官方帳號連結
- IG 連結

### 建議補齊

- 每款產品一句核心描述
- 品牌故事 80-120 字版本
- 出貨與付款說明
- 常見問題 3-5 題

### 若沒有也可先用暫定內容

- 業主本人照片
- 情境照
- 見證內容
- 更完整品牌故事

---

## 7. 設計原則

### 第一版視覺方向

- 手機優先
- 留白充足
- 不做複雜轉場
- 用色對應三款產品，但整體仍要統一
- 保持品牌感，不要像蝦皮賣場

### 技術原則

- 單一 HTML 為主
- Tailwind CSS 可接受
- JS 只保留必要互動
- 盡量避免外部重型動畫依賴
- 頁面載入速度優先

### 對現有 `index-v16.html` 的處理建議

- 可參考其文案語氣與品牌氛圍
- 不直接沿用重動畫與重展示架構
- 第一版應改為更簡潔的轉換導向版

- 沒有明顯假承諾或過度療效暗示
