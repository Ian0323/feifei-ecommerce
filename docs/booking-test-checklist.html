非非768 預約系統 — 測試清單

請依以下項目逐一測試，於空白方框打勾。如遇問題請截圖回報。

一、測試裝置（建議至少各一台）

  • iPhone（用 LINE 內建瀏覽器開）
  • iPhone（用 Safari 開）
  • Android 手機（用 LINE 內建瀏覽器開）
  • Android 手機（用 Chrome 開）
  • 電腦桌機（Chrome / Edge）

二、入口連結（兩個都要測）

  • LIFF 連結：https://liff.line.me/2009821980-46y6BEMh
  • 直連網址：（請填入 GitHub Pages 的 booking.html 網址）
兩個都要驗證：能順利打開，且功能正常。

三、頁面開啟檢查

  • 頁面能正常顯示，沒有破版、亂碼、跑版
  • 最上方有「預約須知」深藍色框 + 同意勾選框
  • 中間顯示「諮詢服務 NT$3,800」
  • 下方有日曆，今天日期是 5/X 時應顯示 5 月

四、預約須知與同意框

  • 沒勾選同意就點日期 → 跳出 alert「請先勾選『我已閱讀預約須知』」，且勾選框會閃高亮
  • 勾選後再點日期 → 順利進到時段選擇
  • 進到時段（步驟 2）後，預約須知整塊收起來不再顯示
  • 按「← 改日期」回到步驟 1，預約須知仍維持收起狀態（不會再跑出來）

五、日曆功能

  • 灰色（不可點）的日期：週六、週日
  • 顯示「滿」字樣的日期：當日 7 個時段全被預約（這項較難測，可略過）
  • 「← 上個月」按鈕 → 在當月開啟時應該是灰色不可點
  • 「下個月 →」按鈕 → 點擊可切到下個月
  • 每次切換步驟（1↔2↔3） → 不會跳回頁面最頂端，而是滑到該步驟區塊

六、時段選擇

  • 一般日（週一～五）顯示 6 個時段：09:00、10:00、11:00、14:00、15:00、16:00
  • 週二多顯示一個 17:00（共 7 個時段）
  • 已被預約的時段：灰色不可點
  • 點「← 改日期」可回到日曆

七、填寫資料表單

各欄位逐一測試錯誤輸入會被擋下：

  • 手機號碼：填「12345」應跳「請輸入有效的手機號碼」
  • Email：填「abc」（無 @）應跳「請輸入有效的 Email」
  • 生年月日：
    • 只剩「民國」，不再有國曆切換選項
    • 年份 placeholder 顯示「民國年」
    • 三格都填正確（例：80 / 5 / 12）→ 通過
    • 任一格留空 → 應該擋下
  • IG 帳號：必填，留空擋下
  • 諮詢主題：感情 / 工作 / 其他
  • 諮詢方式：
    • 選「視訊」→ 顯示視訊匯款資訊
    • 選「現場」→ 顯示現場付款方式
  • 匯款末五碼：必填

八、送出預約

  • 全部填正確 → 跳出確認資訊（可再次檢查）
  • 確認送出 → 顯示「預約成功」頁面，並顯示日期 + 時段
  • 馬上重新整理頁面 → 看時段是否變灰（代表後台已寫入）

九、通知收件確認（這項要等收到通知再回報）

  • 填寫的 Email 信箱有收到預約確認信
  • 如果是從 LINE 連結進來的 → LINE 有收到 feifei 推播訊息
  • 如果是用直連網址（沒登入 LINE）→ LINE 不會推，但 Email 要有

十、錯誤情境

  • 同一個 Email 在 60 秒內連送兩次 → 第二次應被擋下
  • 故意把網路關掉再送出 → 應該顯示錯誤頁，可重試
