> **定位：** 個人品牌帶貨（宗教 IP 方向），長期經營

## 1. 商業模式

### 1.1 生意的本質

feifei 是**個人品牌帶貨**，不是品牌電商。

```
品牌電商：消費者信任「品牌」→ 買產品（光速火箭、innisfree）
個人帶貨：消費者信任「人」  → 買她推薦的東西（feifei）
```

核心資產是 **feifei 這個人的信任度**，不是某個網站或商店。
所有策略都圍繞「如何累積和變現這份信任」來設計。

### 1.2 營收模型

根據市場數據建立的估算：

| 參數 | 數值 | 來源 |
|------|------|------|
| 台灣香氛市場規模 | ~200 億台幣，年增 6.4% | TechNews 光速火箭報導 |
| 中價帶香氛定價 | NT$1,000-3,000 | 光速火箭年銷 44 萬瓶、營收 7 億 |
| 電商官網轉換率基準 | 1-3% | 91APP 產業報告 |
| 美妝/香氛首次獲客成本（投廣告） | 可能 >NT$750/人 | 電商 Tony 陳 |
| 留存率提升 5% 的利潤影響 | +25% 到 +95% | bebit LTV 研究 |
| 開發新客 vs 維護舊客成本比 | 5-7 倍 | 同上 |

**保守估算（純靠 IG 自然流量，不投廣告）：**

```
假設 1,500 IG 粉絲 × 5% 點進官網 = 75 次造訪/月
75 × 2% 轉換率 = ~1.5 筆訂單/月
1.5 × NT$1,200 客單價 = NT$1,800/月
```

**結論：純靠自然流量 + 官網，初期營收很低。這代表：**

1. 不能走「砸廣告獲客」路線（資本不夠，獲客成本會吃掉利潤）
2. 必須走「內容獲客 + 複購養成」路線（獲客成本趨近零）
3. 初期成長會慢，要有耐心
4. feifei 持續產出內容是唯一免費的流量來源——**沒有內容就沒有流量，再好的網站都沒用**

### 1.3 成長引擎

```
適合 feifei 的路線：
  內容（IG）→ 信任 → 首單 → 複購 → 口碑傳播

  優勢：獲客成本趨近零
  挑戰：成長速度取決於內容品質和產出頻率

不適合 feifei 的路線：
  砸廣告 → 獲客 → 首單微虧 → 靠複購賺回來

  問題：需要資本墊付，feifei 目前規模不適合
```

### 1.4 美妝/香氛品類特性

根據電商 Tony 陳的分類，香氛屬於**「好賺不好賣」**：

- 毛利率高（自有品牌可達 60-70%）
- 但獲客難、競爭激烈
- 大品牌靠資本燒廣告，小品牌靠不了
- **小品牌的出路：個人 IP 信任 + 內容行銷 + 複購經營**

光速火箭的成功印證了這一點——即使年營收 7 億，行銷費用也控制在銷售額 25% 以下。

## 2. 行銷策略

### 2.1 策略分層

#### 第一層：短期（0-3 個月）— 從 0 到穩定出單

**目標：每月穩定 10+ 筆訂單**

| 動作 | 為什麼 | 誰負責 |
|------|--------|--------|
| feifei 持續產出 IG 內容 | 唯一免費的流量來源 | feifei 產出，Ian 協助方向 |
| 每則內容都帶購買連結 | 減少「想買但找不到入口」的流失 | Ian 確保連結正確 |
| 金流即時化 | 客人點了就能付，不需等回覆 | Ian 串接綠界 |
| 建立 LINE 社群 | 把 IG 一次性流量轉成可反覆觸及的私域流量 | 小非 AI + feifei |
| 收集第一批客戶真實回饋 | 真實見證 > 任何行銷文案 | feifei 收集，Ian 整理上架 |

#### 第二層：中期（3-6 個月）— 提升複購和口碑

**目標：讓買過的人再買、帶朋友買**

| 動作 | 為什麼 | 數據支撐 |
|------|--------|---------|
| 複購提醒（用完時間到了推播） | 留存率 +5% → 利潤 +25-95% | bebit LTV 研究 |
| 組合包 / 三瓶套組 | 拉高客單價 | 光速火箭類似策略 |
| 口碑分享機制 | 開發新客成本是維護舊客的 5-7 倍 | LTV 研究 |
| 內容模板化 | 讓 feifei 更高效產出 | AI 輔助文案/排程 |

#### 第三層：長期（6-12 個月）— 擴展產品線和通路

- 視香霧營收穩定度決定是否開 WACA 選物商店
- 視 LINE 群組規模決定是否做更多自動化
- 視宗教內容互動數據決定是否深化宗教 IP 定位

### 2.2 宗教 IP 定位：待驗證

業主目前的方向是走宗教 IP，但市場調研顯示：

- **找不到**台灣宗教 KOL 帶貨的成功案例（可能是藍海，也可能是偽命題）
- 農民曆 APP 已有大量競品（科技紫微網、Google Play 上十幾款）
- 宮廟信眾的數位消費行為缺乏公開數據

**策略：先讓 feifei 在 IG 測試宗教相關內容的互動率，用數據決定是否 all-in。**

具體做法：
1. feifei 產出 5-10 則宗教相關 IG 內容（拜拜知識、農曆提醒、開運分享）
2. 同時產出 5-10 則非宗教內容（生活風格、產品使用場景）
3. 比較兩類內容的互動率（按讚、留言、分享、連結點擊）
4. 數據顯示宗教內容明顯好 → 加碼宗教方向，開始做系統化功能
5. 數據沒有差異 → 維持混合路線，不做額外功能投資

## 4. 金流策略

### 4.1 核心原則

**客人看到就能付，不需要加 LINE 諮詢、不需要等回覆。**

### 4.2 分階段方案

| 階段 | 金流方式 | 適用時機 |
|------|---------|---------|
| 初期（月 0-20 筆） | Google 表單 + 匯款末五碼 | 成本 $0，feifei 手動對帳 |
| 成長期（月 20+ 筆） | 綠界 ECPay 收銀台 | 手續費 1.68-2.75%，自動化 |
| 擴展期 | WACA PayNow | 選物商店上線時 |

**切換信號：feifei 開始覺得對帳很煩 → 就是該切綠界的時候。**

綠界收銀台（非 API 串接）：個人可申請、免月租、免寫程式，在後台建商品產生付款連結即可。

### 4.3 LINE 的定位

LINE **不是訂單入口**：
- 社群經營（群組互動、話題）
- 客服（售後問題、轉人工）
- 導流（引導去官網完成付款）

## 5. 技術架構

### 5.1 當前架構

```
  GitHub Pages 官網（已上線）
  ├── 產品頁（三款香霧 + 付款 CTA）
  └── products.json（資料驅動）

  LINE Bot（已開發，待部署）
  ├── 一對一私訊 → FAQ + 產品推薦 + Claude AI
  ├── 群組互動   → 節流 + 導購
  └── 排程推播   → 待開發

  外部服務（待串接）
  ├── 綠界 ECPay  ← 香霧即時付款
  ├── GA4        ← 網站流量追蹤
  └── Claude API ← 小非 AI 回覆
```

### 5.2 技術原則

- 靜態前端（HTML + Tailwind + 少量 JS），GitHub Pages 部署
- 資料驅動：內容從 JSON 讀取，更新 JSON 即更新網站
- LINE Bot 用 Node.js + Express（已有骨架）
- 不做前端框架，保持簡單可維護
- 不自建資料庫
- **功能只在有數據支撐時才加，不預先蓋空的區塊**

## 7. 導流路徑

### 主路徑：香霧購買

```
IG 限動 / Reels / Bio
  → 官網產品頁
  → 看三款差異
  → 點「立即購買」
  → 付款頁（即時完成付款）
  → 業主收到通知 → 出貨
```

**關鍵：從看到產品到完成付款，不需要離開流程。**

### 社群路徑：LINE

```
LINE 群組 / 一對一
  → 小非互動 / 話題討論
  → 想買的人 → 導去官網產品頁付款
  → 售後問題 → 轉 feifei 本人
```

## 11. AI 執行規範

### 11.1 核心原則

1. **先上線，再完善** — 有東西比完美重要
2. **即時付款** — CTA 是「立即購買」，不是「加 LINE 洽詢」
3. **LINE 不處理訂單** — LINE 是社群，不是收銀台
4. **數據驅動** — 功能只在有數據支撐時才做，不靠直覺
5. **先驗證再開發** — 不預先蓋空的區塊，等有真實需求再做
6. **內容是引擎** — feifei 的內容產出是一切的前提

### 11.2 每次開發前

- 先讀本 `plan.md`
- 再看 `PROGRESS.md` 確認當前進度
- 新功能必須回答：「有什麼數據支撐這個功能值得做？」
- 完成後更新 `PROGRESS.md`

### 11.3 文案原則

- 語氣溫柔但直接，像主理人在推薦自己真的會用的東西
- CTA 是「立即購買」不是「加 LINE 洽詢」
- 不承諾療效、不編造故事
- 宗教/信仰元素的比重依 IG 數據驗證結果調整

### 11.4 LINE Bot 原則

- 不假裝真人
- 不處理訂單和付款
- 想買的人一律導去官網付款連結
- 敏感問題導回 feifei 本人
