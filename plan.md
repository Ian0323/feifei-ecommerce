## 2. 金流策略

### 2.1 核心原則

**客人看到就能付，不需要加 LINE 諮詢、不需要等回覆。**

### 2.2 金流方案

| 產品線 | 金流方式 | 說明 |
|--------|---------|------|
| 水晶香霧 | 綠界 ECPay 或 WACA 內建金流 | 官網產品頁直接結帳 |
| 非非選物 | WACA PayNow | 商店內結帳 |
| Google 表單 | 最低限度備案 | 僅在金流尚未串接時暫用 |

### 2.3 金流串接優先順序

1. **Phase 1（先上線）：** 產品頁 CTA → 綠界付款連結（每個商品一個獨立付款連結，免開發即可用）
2. **Phase 2（穩定後）：** 如果量大再考慮 WACA 統一管理或自建簡易購物車
3. **備案：** Google 表單 + 匯款（僅在綠界/WACA 尚未就緒時使用）

### 2.4 LINE 的定位

LINE **不是訂單入口**。LINE 的角色：
- 社群經營（群組互動、話題、見證分享）
- 農曆推播（初一十五、財神日提醒）
- 客服（售後問題、轉人工）
- 導流（引導去官網看產品、完成付款）

## 4. 產品線規劃

### 4.1 水晶香霧（主力產品）

- 三款：淨化、貴人、招財
- 更動頻率低，適合固定產品頁
- 官網直接結帳（綠界付款連結）
- 搭配農曆行事曆做推薦（財神日推招財、沖煞日推淨化）

### 4.2 非非選物（擴展產品線）

- 韓國服飾 / 選物
- 更動頻率高，需要上下架後台
- WACA 商店管理
- 上線時間：待香霧產品線穩定後

### 4.3 未來可能（暫不開發）

- 線上課程（拜拜知識、開運方法）
- 宮廟活動團購
- 聯名商品

### 5.2 技術選擇原則

- 靜態前端（HTML + Tailwind + 少量 JS），部署在 GitHub Pages

- 不做前端框架（React/Vue），保持業主可維護
- 不自建資料庫，用 JSON 檔 + 第三方服務

## 7. 導流路徑

```
  IG 限動 / Reels / Bio
    → link-in-bio
    → 官網產品頁
    → 看三款差異
    → 點「立即購買」
```

```
  IG Bio / 限動
    → link-in-bio
    → WACA 商店
    → 瀏覽 + 加購物車
    → 結帳（PayNow 金流）
```

### 7.3 路徑 C：社群經營（LINE）

```
  LINE 群組 / 一對一
    → 小非互動 / 農曆推播 / 話題討論
    → 想買的人 → 導去官網產品頁付款
    → 售後問題 → 轉 feifei 本人
```

**LINE 是養信任的地方，不是處理訂單的地方。**

## 11. AI 執行規範

### 11.1 核心原則

1. **先上線，再完善** — 有東西比完美重要
2. **即時付款優先** — 不讓客人「先問再買」，要「看到就能買」
3. **LINE 不處理訂單** — LINE 是社群，不是收銀台
4. **宗教內容要真誠** — 不編造靈驗故事，不過度承諾效果
5. **資料驅動** — 所有內容從 JSON 讀取，更新 JSON 即更新全站
6. **長期思維** — 每個決策都要考慮 6 個月後的影響

### 11.2 每次開發前

- 先讀本 `plan.md`
- 再看 `PROGRESS.md` 確認當前進度
- 若要新增工作項，先補到本文件再執行
- 完成後更新 `PROGRESS.md`

### 11.3 文案原則

- 宗教場景為主，身心靈質感為輔
- 語氣溫柔但直接，不繞彎
- CTA 是「立即購買」不是「加 LINE 洽詢」
- 不承諾療效、不編造靈驗故事
- 可以提農曆、拜拜、財神，但不要過度玄學

### 11.4 LINE Bot 原則

- 不假裝真人
- 不處理訂單和付款
- 想買的人一律導去官網付款連結
- 敏感問題導回 feifei 本人
- 不編造產品功效
